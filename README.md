# Гороскоп на сегодня

Готовая статическая веб-страница для размещения на GitHub Pages и запуска на iOS / Android.

## Что умеет

- показывает гороскоп на текущий день по дате устройства;
- поддерживает 12 знаков зодиака;
- запоминает выбранный знак;
- работает без серверной части и баз данных;
- адаптирована под смартфоны, планшеты и горизонтальные дисплеи;
- поддерживает PWA-режим: можно добавить на главный экран iOS / Android;
- после первого открытия кэшируется service worker и может открываться быстрее.

## Как разместить на GitHub Pages

1. Создайте новый репозиторий на GitHub, например `daily-horoscope`.
2. Загрузите в репозиторий все файлы из этого архива:
   - `index.html`
   - `styles.css`
   - `app.js`
   - `analytics-config.js`
   - `analytics.js`
   - `manifest.webmanifest`
   - `sw.js`
   - папку `icons`
3. Откройте настройки репозитория: `Settings` → `Pages`.
4. В разделе `Build and deployment` выберите:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Сохраните настройки.
6. Через некоторое время GitHub покажет ссылку вида:
   `https://ВАШ_ЛОГИН.github.io/daily-horoscope/`

## Как запустить на iOS

1. Откройте ссылку GitHub Pages в Safari.
2. Нажмите кнопку «Поделиться».
3. Выберите «На экран Домой».
4. Запускайте как обычное приложение.

## Как запустить на Android

1. Откройте ссылку GitHub Pages в Chrome.
2. Нажмите меню ⋮.
3. Выберите «Добавить на главный экран» или «Установить приложение».
4. Запускайте с главного экрана.


## Статистика посещений и действий пользователей

В проект добавлен модуль аналитики:

- просмотры страниц;
- посещения и уникальные пользователи — в интерфейсе выбранного сервиса аналитики;
- устройства, браузеры, страны/города, источники переходов — зависит от выбранного сервиса;
- события внутри страницы:
  - `forecast_view` — просмотр прогноза;
  - `sign_select` — выбор знака зодиака;
  - `forecast_refresh` — обновление прогноза;
  - `share` — нажатие «Поделиться»;
  - `install_hint_open` — открытие инструкции установки на экран;
  - `fullscreen_open` — запуск во весь экран.

### Как включить аналитику

Яндекс Метрика уже включена в файле `analytics-config.js` со счетчиком `110547743`.

#### Вариант 1. Google Analytics 4

1. Создайте ресурс GA4.
2. Скопируйте Measurement ID вида `G-XXXXXXXXXX`.
3. В `analytics-config.js` установите:

```js
googleAnalytics: {
  enabled: true,
  measurementId: "G-ВАШ_ИДЕНТИФИКАТОР"
}
```

#### Вариант 2. Яндекс Метрика

1. Создайте счетчик Яндекс Метрики.
2. Скопируйте номер счетчика.
3. В текущем проекте уже установлено:

```js
yandexMetrica: {
  enabled: true,
  counterId: 110547743,
  clickmap: true,
  trackLinks: true,
  accurateTrackBounce: true,
  webvisor: false
}
```

Для событий можно создать цели типа «JavaScript-событие» с идентификаторами `forecast_view`, `sign_select`, `forecast_refresh`, `share`, `install_hint_open`, `fullscreen_open`.

#### Вариант 3. Umami

Подходит, если нужна более простая и приватная аналитика, в том числе с самостоятельным размещением на VPS.

```js
umami: {
  enabled: true,
  websiteId: "ВАШ_WEBSITE_ID",
  scriptUrl: "https://cloud.umami.is/script.js"
}
```

### Важное ограничение GitHub Pages

GitHub Pages размещает статические файлы. Сам проект не хранит базу клиентов и не ведет серверные журналы посещений, доступные владельцу сайта. Поэтому статистика собирается через внешний счетчик или через отдельный backend/API.

### Персональные данные

Текущая реализация не запрашивает ФИО, телефон, email и другие персональные данные. Баннер предупреждения о сборе статистики удален по требованию. Если планируется публичный запуск или сбор персональных данных клиентов, рекомендуется отдельно проверить требования законодательства и добавить политику конфиденциальности.

## Важно

Прогноз генерируется локально по дате и знаку зодиака. Это развлекательный сервис, он не использует платные API и не требует интернета после первичной загрузки.
