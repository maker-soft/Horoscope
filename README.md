# Мультигороскоп для GitHub Pages

Статический сайт для GitHub Pages с несколькими видами гороскопов:

- гороскоп на день по знаку зодиака;
- гороскоп на месяц по знаку зодиака;
- китайский гороскоп по году рождения;
- гороскоп совместимости пары.

## Что изменено в этой версии

1. Добавлен первый экран с выбором вида гороскопа.
2. Для гороскопа на день и на месяц используется существующая страница выбора знака зодиака с кругом знаков.
3. В круге знаков Козерог расположен сверху.
4. На странице выбора знака добавлены только две кнопки периода:
   - «На день»;
   - «На месяц».
5. Гороскоп по китайскому календарю работает через выбор года рождения.
6. Совместимость рассчитывается по отдельным данным мужчины и женщины:
   - знак зодиака мужчины;
   - год рождения мужчины по китайскому календарю;
   - знак зодиака женщины;
   - год рождения женщины по китайскому календарю.
7. Яндекс Метрика сохранена и включена: `counterId = 110547743`.
8. Баннер предупреждения о сборе статистики отсутствует.
9. Добавлена локальная база месячных прогнозов: `12 месяцев × 12 знаков = 144 варианта`.
10. Расширен китайский гороскоп: добавлен файл `chinese-forecasts-data.js` с `365` ежедневными вариантами советов.
11. Исправлена адаптивная верстка всех разделов: текст не накладывается, длинные блоки корректно прокручиваются внутри экрана.
12. На всех рабочих экранах добавлен переход назад или на стартовый экран.

## Структура файлов

- `index.html` — структура приложения и все экраны.
- `styles.css` — адаптивный дизайн для iOS, Android и десктопа.
- `app.js` — логика навигации, выбора знаков, года рождения и совместимости.
- `forecasts-data.js` — локальная база ежедневных прогнозов: 365 дней × 12 знаков = 4 380 вариантов.
- `monthly-forecasts-data.js` — локальная база месячных прогнозов: 12 месяцев × 12 знаков = 144 варианта.
- `chinese-forecasts-data.js` — локальная база китайского гороскопа: 365 вариантов ежедневных советов.
- `analytics-config.js` — настройки аналитики.
- `analytics.js` — отправка событий в аналитику.
- `manifest.webmanifest` — настройки PWA.
- `sw.js` — кэширование статических файлов.
- `icons/` — иконки приложения.

## События аналитики

В Яндекс Метрику передаются события:

- `app_start` — запуск приложения;
- `menu_select` — выбор вида гороскопа;
- `screen_view` — просмотр экрана;
- `period_mode_select` — выбор периода день/месяц;
- `sign_select` — выбор знака зодиака;
- `forecast_view` — просмотр прогноза по знаку;
- `chinese_forecast_view` — просмотр китайского гороскопа;
- `compatibility_view` — расчет совместимости;
- `share` — нажатие «Поделиться».

## Размещение на GitHub Pages

1. Создайте репозиторий на GitHub.
2. Загрузите все файлы из архива в корень репозитория.
3. Откройте `Settings` → `Pages`.
4. Выберите публикацию из ветки `main`, папка `/root`.
5. Откройте ссылку GitHub Pages после публикации.

Сайт не требует backend-сервера и работает как статическая HTML/CSS/JS-страница.
