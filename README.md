# Гороскоп на сегодня — версия с первым экраном выбора знака

Статический сайт для размещения на GitHub Pages. Первый экран полностью отведен под круг выбора знака зодиака. Знаки расположены по кругу, начиная с Козерога в верхней точке. После выбора знака в этом же окне открывается экран результата гороскопа, а круг знаков скрывается.

## Что изменено в этой версии

- Первый экран: полноэкранный круг выбора знака зодиака.
- Верхняя позиция круга: Козерог.
- После выбора открывается отдельный экран результата без отображения круга знаков.
- Добавлена кнопка `← Знаки` для возврата к выбору знака.
- Убраны блоки и элементы: `Другой вариант`, `Цвет`, `Число`, технический код варианта прогноза.
- Сохранена локальная база прогнозов: 365 дней × 12 знаков = 4 380 вариантов.
- Сохранена Яндекс Метрика с номером счетчика `110547743`.

## Состав файлов

- `index.html` — основная страница.
- `styles.css` — адаптивный дизайн двух экранов.
- `app.js` — логика выбора знака и показа результата.
- `forecasts-data.js` — локальная база прогнозов.
- `analytics-config.js` — настройки аналитики.
- `analytics.js` — подключение Яндекс Метрики / GA4 / Umami.
- `manifest.webmanifest`, `sw.js`, `icons/` — вспомогательные файлы веб-приложения.

## Аналитика

В проекте включена Яндекс Метрика:

```js
counterId: 110547743
```

Отправляемые события:

- `select_screen_view` — открыт экран выбора знака;
- `sign_select` — выбран знак зодиака;
- `forecast_view` — открыт экран гороскопа;
- `sign_change_open` — возврат к выбору знака;
- `share` — нажатие кнопки «Поделиться».

## Размещение на GitHub Pages

1. Создайте новый репозиторий на GitHub.
2. Загрузите все файлы из архива в корень репозитория.
3. Откройте `Settings` → `Pages`.
4. Выберите `Deploy from a branch`.
5. Укажите ветку `main` и папку `/root`.
6. Откройте опубликованную ссылку GitHub Pages.
